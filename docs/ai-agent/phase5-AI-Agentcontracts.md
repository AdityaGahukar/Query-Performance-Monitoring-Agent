## 1. Class Interface Contract

The Analysis Engine is exposed to the aggregator orchestrator via the following service interface:

```python
class QueryPerformanceAnalyzer:
    """
    Orchestrates telemetry prompt formatting, provider invocation, 
    structured output parsing, validation, and metadata collection.
    """

    def __init__(self, provider: LLMProvider, settings: Settings):
        self.provider = provider
        self.settings = settings

    def analyze_finding(
        self,
        snapshot: TelemetrySnapshot,
        issues: List[DetectedIssue],
        operator_stats: Optional[List[Dict[str, Any]]] = None
    ) -> AnalysisResult:
        """
        Main entry point for evaluating a detected performance finding.
        
        Args:
            snapshot: The point-in-time telemetry snapshot of the query.
            issues: The deterministic issues triggered by the detector.
            operator_stats: Optional execution plan operators (unpruned).
            
        Returns:
            A validated AnalysisResult instance matching the Pydantic domain models.
            
        Raises:
            LLMProviderError: If the LLM provider experiences network or API failures.
            LLMTimeoutError: If the LLM call times out.
            LLMValidationError: If the LLM output fails schema validation after retries.
        """
        pass
```

## 3. Output Schema & Domain Model Mapping

The output generated by the LLM must match the following structure. The analyzer maps this JSON directly to the Pydantic classes defined in [models.py](file:///Users/as-mac-1299/Intern%20Projects/Snowflake%20Performance%20Monitoring/src/domain/models.py).


### Domain Models Definitions

```python
class ConfidenceScore(BaseModel):
    score: float = Field(..., ge=0.0, le=1.0)
    reason: str = Field(..., min_length=1)

    model_config = {"frozen": True}

class Recommendation(BaseModel):
    recommendation_id: UUID = Field(default_factory=uuid4)
    recommendation_type: str = Field(..., min_length=1)
    description: str = Field(..., min_length=1)
    expected_impact: str = Field(..., min_length=1)
    priority: str = Field(..., min_length=1)
    rationale: str = Field(..., min_length=1)
    evidence: str = Field(..., min_length=1)

    model_config = {"frozen": True}

class AnalysisResult(BaseModel):
    analysis_id: UUID = Field(default_factory=uuid4)
    root_cause_summary: str = Field(..., min_length=1)
    recommendations: list[Recommendation] = Field(..., min_length=1)
    llm_metadata: dict[str, Any] = Field(...)
    confidence: ConfidenceScore = Field(...)

    model_config = {"frozen": True}
```

## 5. Exceptions & Error Recovery State

In the event of structural errors, the orchestrator catches downstream exceptions and routes the finding to the DLQ table to ensure processing of future telemetry batches continues.

### Exception Hierarchy

* **`LLMAnalysisError(Exception)`**: Base exception for all Analysis Engine errors.
* **`LLMProviderError(LLMAnalysisError)`**: Triggered when the service provider returns non-200 responses, network timeouts occur, or rate-limits are breached.
* **`LLMTimeoutError(LLMAnalysisError)`**: Raised if the provider invocation duration exceeds the configured connection timeout budget.
* **`LLMValidationError(LLMAnalysisError)`**: Raised when the response JSON cannot be parsed or lacks required Pydantic fields after retry attempts.

### Fallback Alert Flow (DLQ)

If an analysis fails permanently, the aggregator saves the `PerformanceFinding` with `analysis = None`, creates an `AlertEvent` with `status = FAILED`, and writes it to `POV4_DLQ`:

```json
{
  "ALERT_ID": "e0e84b16-43d6-444a-8d05-248c8bfaef68",
  "FINDING_REFERENCE": "finding_uuid_here",
  "DESTINATION": "POV3_WEBHOOK",
  "STATUS": "FAILED",
  "DELIVERY_TIMESTAMP": "2026-06-24T12:00:00Z"
}
```
The alert payload sent to the DLQ contains details about the failed analysis, allowing system operators to perform manual reviews or trigger diagnostic backfills.
