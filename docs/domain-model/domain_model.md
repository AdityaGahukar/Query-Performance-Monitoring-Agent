# Domain Model Design

This document explicitly defines the core entities in the POV-4 domain model.

## Core Entities

### 1. TelemetrySnapshot
**Responsibility**: Represents a point-in-time capture of Snowflake workload metrics. It holds the raw data that will be evaluated for performance bottlenecks.
**Attributes**:
- `snapshot_id` (UUID)
- `timestamp` (DateTime)
- `query_id` (String)
- `warehouse_name` (String)
- `execution_time_ms` (Integer)
- `queued_overload_time_ms` (Integer)
- `bytes_spilled_to_local_storage` (Integer)
- `bytes_spilled_to_remote_storage` (Integer)
- `query_profile_data` (JSON)
**Usage**: Ingested by the Data Collection Layer and passed to the Monitoring Engine.

### 2. DetectedIssue
**Responsibility**: Represents a specific performance bottleneck identified by the deterministic rule-based engine.
**Attributes**:
- `issue_id` (UUID)
- `type` (Enum: REMOTE_SPILL, LOCAL_SPILL, WAREHOUSE_QUEUING, LONG_RUNNING)
- `severity` (Enum: LOW, MEDIUM, HIGH, CRITICAL)
- `threshold_breached` (String)
- `actual_value` (Float)
- `telemetry_reference` (UUID -> TelemetrySnapshot)
**Usage**: Generated by the Issue Detection Engine and passed to the Performance Analysis Agent.

### 3. Recommendation
**Responsibility**: An actionable step to mitigate or resolve a performance issue, generated by the AI agent.
**Attributes**:
- `recommendation_id` (UUID)
- `action_type` (Enum: SCALE_UP, CLUSTER_TABLE, OPTIMIZE_SQL)
- `description` (String)
- `expected_impact` (String)

### 4. AnalysisResult
**Responsibility**: The output from the LLM containing root cause analysis and a list of recommendations.
**Attributes**:
- `analysis_id` (UUID)
- `root_cause_summary` (String)
- `recommendations` (List[Recommendation])
- `confidence_score` (Float, 0.0 - 1.0)
- `confidence_reason` (String)
- `llm_metadata` (JSON - latency, model version, token usage)
**Usage**: Output of the Performance Analysis Agent.

### 5. PerformanceFinding
**Responsibility**: The aggregate, persisted record that acts as the system's source of truth. Consumed by downstream systems (POV-3).
**Attributes**:
- `finding_id` (UUID)
- `timestamp` (DateTime)
- `query_id` (String)
- `warehouse` (String)
- `overall_severity` (Enum)
- `issues` (List[DetectedIssue])
- `metrics` (TelemetrySnapshot)
- `analysis` (AnalysisResult)
**Usage**: Stored in the Performance Findings Store, sent as a payload in AlertEvent.

### 6. AlertEvent
**Responsibility**: Represents a notification trigger based on a PerformanceFinding.
**Attributes**:
- `alert_id` (UUID)
- `finding_reference` (UUID -> PerformanceFinding)
- `destination` (Enum: TEAMS, EMAIL, POV3_WEBHOOK)
- `status` (Enum: PENDING, SENT, FAILED)
- `delivery_timestamp` (DateTime)
**Usage**: Handled by the Notification Service and POV-3 Integration Layer.
